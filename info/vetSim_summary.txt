 vetsim.h — The Central Data Structure Header                                                                                             
  This file defines all the shared data structures that WinVetSim uses to manage simulation state. Think of it as the "schema" for the   
  entire simulation.

  Key Architecture

  WinVetSim uses a shared memory pattern with two main copies of simulation data:

  1. struct status (line 334) — The current state of the simulation. Read by the browser (via JSON) and sim-ctl (via TCP). This is the   
  "truth" of what the sim is doing right now.
  2. struct instructor (line 355) — The command inbox. When the instructor interface (sim-ii) sends a set: command, it gets written here.
   Fields are initialized to -1 meaning "no command pending."

  Both structs contain the same sub-structs (cardiac, respiration, eyes, etc.), but they serve different purposes.

  Data Flow Through the Structs
  sim-ii sends:  set:eyes:right_pupil=50
                      |
                      v
     simstatus.cpp parses -> eyes_parse() writes to instructor.eyes.right_pupil = 50
                      |
                      v
     VetSim.cpp scan_commands() sees instructor.eyes.right_pupil >= 0
        -> copies to status.eyes.right_pupil = 50
        -> resets instructor.eyes.right_pupil = -1  (consumed)
                      |
                      v
     sendStatus() reads status.eyes.right_pupil -> JSON to browser

  The struct eyes Addition (lines 309-325)
  struct eyes
  {
      int connected;       // 1 = RP2040 found on I2C, 0 = not found
      int right_state;     // 0=Normal, 1=Obtunded, 2=Miotic, 3=Dilated
      int right_lid;       // 0=Open, 1=Closed, 2=Partial
      int right_move;      // 0=Normal, 1=InfreqSlow, 2=None
      int right_position;  // 0-8 (center, cardinal, diagonal)
      int right_blink;     // 0=Normal, 1=InfreqSlow, 2=PartialInfreq, 3=None
      int right_pupil;     // 5-90 (percent)
      int left_state;      // Same as right
      int left_lid;
      int left_move;
      int left_position;
      int left_blink;
      int left_pupil;
      int send_command;    // Trigger to send accumulated values to RP2040
  };

  This was added after struct vocals (line 301-308), and then embedded into both:
  - struct status at line 350: struct eyes eyes;
  - struct instructor at line 367: struct eyes eyes;

  The prototype for the parser was added at line 508:
  int eyes_parse(const char* elem, const char* value, struct eyes* eyes);

  VetSim.cpp — scan_commands() Eyes Section (lines 1957-2027)

  This is the command processor. It runs periodically and transfers pending commands from instructor to status. The pattern for every    
  field is identical:
  if (simmgr_shm->instructor.eyes.right_pupil >= 0)       // Is there a pending command?
  {
      simmgr_shm->status.eyes.right_pupil =                // Copy to status (now "live")
          simmgr_shm->instructor.eyes.right_pupil;
      simmgr_shm->instructor.eyes.right_pupil = -1;        // Reset instructor (-1 = "consumed")
  }

  This repeats for all 14 eyes fields (connected, right_state, right_lid, right_move, right_position, right_blink, right_pupil, and the  
  same 6 for left, plus send_command).

  How Eyes Differs From Other Subsystems

  Most other subsystems (cardiac, respiration, temperature) use trends — gradual transitions over time via setTrend(). For example, heart
   rate changes smoothly over a transfer_time period.

  The eyes code does direct copy — when you set right_pupil=50, it immediately becomes 50 in status. No trend/transition. This makes     
  sense because eye changes (pupil dilation, lid position) should be instantaneous in the simulation.

  Where Eyes Fits in scan_commands()

  The processing order in scan_commands() is:
  1. Cardiac (heart rate, rhythm, blood pressure, etc.)
  2. Respiration (lung sounds, SPO2, ETCO2, etc.)
  3. General (temperature)
  4. Media/Vocals
  5. Telesim
  6. Eyes (line 1957)
  7. CPR
  8. Defibrillation

  The eyes block sits between telesim and CPR — a natural location since telesim and eyes are both "peripheral" features added to the    
  base sim.
